## 커밋 메시지 작성 규칙

- 커밋 메시지는 **의도 중심으로 명확하게 작성**해주세요.
- 기본 포맷: `<타입>: <무엇을 변경했는지 요약>`
- 패키지 설치 시: 설치한 패키지와 목적을 적어주세요

-  예시:
  `✨Feat: 로그인 폼 UI 구현`  
  , `🐛Fix: 결제 실패 오류 수정`

## 체크리스트
- [feat]: 기능 추가
- [fix]: 버그 수정
- [docs]: 문서 수정
- [refactor]: 리팩토링 수정
- [test]: 테스트 결과
- [chore]: 설정/의존성/기타 등
- [perf]: 성능 개선
